## 端口定义

端口以 **spec §2.3「M2 packet_sram」表（第 193–204 行）为唯一权威**，DE 按该表逐信号实现，方向/位宽/命名不得偏离。8 个信号：

| 方向 | 信号 | 位宽 | 来源/去向 |
| --- | --- | --- | --- |
| 输入 | clk | 1 | ppa_top.PCLK |
| 输入 | rst_n | 1 | ppa_top.PRESETn 映射（低有效） |
| 输入 | wr_en | 1 | M1 写使能 |
| 输入 | wr_addr | 3 | M1 写地址（0–7） |
| 输入 | wr_data | 32 | M1 写数据 |
| 输入 | rd_en | 1 | M3 读使能 |
| 输入 | rd_addr | 3 | M3 读地址（0–7） |
| 输出 | rd_data | 32 | 送 M3 读数据 |

## 功能要求（对应 feature-matrix）

- **F1-6**（§2.3）：8×32-bit 双端口同步 SRAM——独立写端口（M1）与读端口（M3），存储 8 个 32-bit word。

## 边界与约束（每条标 spec 章节号）

- **深度与位宽（§2.3 §6.1）**：8 深（wr_addr/rd_addr 3-bit，合法 0–7）× 32-bit；不做超出 8 word 的存储（§6.1 窗口共 8 word）。
- **同步时序（§2.2 "双端口同步 SRAM"）**：读端口同拍组合读，`rd_en=1` 当拍 `rd_data=mem[rd_addr]` 有效，无 1 拍寄存延迟；写端口同步写（clk 上升沿按 wr_en 写入）（§2.3 M2 表注 r6、§7.3 第 0 拍说明 r6）。
- **整字存储（§3.3 §6.2）**：每次写入为 32-bit 整字，字节拆分/组织由 M1 完成后以整字送入；本模块按 word 存取，不做字节级拆分或部分字节写。
- **不做写保护仲裁（§6.3 教学提示）**：`busy=1` 期间的 PKT_MEM 写保护由 M1 的 PSLVERR 机制实现；M2 收到的 `wr_en` 即视为有效写，不判 busy、不判地址窗口合法性（M1 已译码）。
- **不做包语义判断（§2.2）**：本模块对写入内容不解析、不校验、不因 pkt_len/pkt_type 改变存储行为；「最后一个 word 部分有效」的判定属 M3 读取阶段（§6.2），M2 仍完整存储 32-bit。
- **复位（§2.3）**：`rst_n` 低有效。spec 未规定 SRAM 阵列复位后的初值语义（存储阵列通常不含复位）；DE 不得凭空对外承诺复位清零行为——存储初值对外可见性属未定义，验证侧只在写后读回比对（§11.2 必2）。此项对外影响小，暂不单列提案，若后续验收需要再走 §8。

> 以上均为 spec 已定义/可直接推得的约束（读时序契约见 BUG-003/r6 已裁决歧义）；DE 不得新增 spec 之外的对外可见行为（复位初值语义等未定项走 §8 提案，不在本文件或 RTL 私定）。

## 内部断言建议（非强制，DE 撰写）

- `wr_addr`、`rd_addr` 恒在 0–7 范围内（3-bit 天然满足，可断言无 X）。
- 同拍/同地址无中间写冲突时，`rd_data` 应等于最近一次写入 `wr_addr` 相同地址的 `wr_data`（读为组合输出，无寄存延迟，§2.3 r6）。
- `wr_en`/`rd_en` 为 X 时不得触发写/读（复位后信号确定性）。

## 已裁决歧义

- **BUG-003 / spec 修改记录 r6（SPEC_CHANGED，rev 裁决 2026-07-09）**：M2 读端口时序契约确定为**同拍组合读**（`rd_en=1` 当拍 `rd_data=mem[rd_addr]` 有效，无 1 拍寄存延迟），写端口维持同步（clk 上升沿）写。锚点：§2.3 M2 表注 r6、§7.3 第 0 拍说明 r6。本模块的"同步时序"约束与"写后读一致"内部断言均据此更新。
- BUG-001（r4，exp_pkt_len 语义）、BUG-002（r5，非法包长 sum/xor 与读拍钳位）均属 M1/M3 行为，与 M2 纯存储职责无关。

## 验收关联（testplan 场景 ID，DE 自检不替代 DV 验证）

- **M1-02**（§6.1 §11.2-必2）：PKT_MEM 写入地址映射——本模块承接的存储行为部分（wr_en/wr_addr/wr_data 写入后可正确读回）。

## 明确不做

- **地址译码 / APB 时序**：`0x040+4×N` 到 Word N 的译码、APB 两段式时序、PSLVERR 均为 M1（apb_slave_if）职责；M2 只接收已译码的 `wr_addr`（§2.2 §6.1 §6.3）。
- **包语义判断 / FSM**：包头解析、格式检查、payload 摘要、读地址递增控制均为 M3（packet_proc_core）职责；M2 不含状态机（§2.2 §7）。
- **busy 写保护仲裁**：由 M1 的 PSLVERR 实现，M2 不参与（§6.3 教学提示）。
