# <模块名> 设计输入

- **里程碑**：M_
- **spec 依据章节**：§_（逐条列出，含第 0 章适配项）
- **端口定义**：引用 spec §2.3 对应表（如有偏离必须先修 spec）
- **功能要求**：对应 feature-matrix 条目编号 + 一句话/条
- **边界与约束**：复位行为、时序约定（如"start 后 1 拍 busy=1"）、错误响应
- **已裁决歧义**：引用 bugs.md 条目号（SPEC_CHANGED）与 spec 修改记录版次
- **验收关联**：testplan 场景 ID（DE 自检不替代 DV 验证）
- **明确不做**：超出本模块职责的内容（如 M2 不做包语义判断）
