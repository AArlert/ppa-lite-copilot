四角色多 Agent 机制

orch（主会话）= 纯指挥家，不产出技术工件；专业工作全部派单

arch 设计输入

spec 修改提案、design-prompt、feature-matrix、接口定义。

禁区：对外可见行为只准进 spec，不得藏在 design-prompt（行为泄漏）；spec 只能提案。

de RTL

按 design-prompt + spec 写 rtl/ 与内部不变量断言；编译 + lint 自检。

禁区：不碰 tb/、testplan。

dv 验证

tb/ UVM 环境、接口/协议 SVA、testplan 场景、仿真与证据。

禁区：checker 只准从 spec 推导，禁止照抄 RTL 行为。

rev 门禁

arch 交付门禁、代码/checker 审查、DE-DV 仲裁、里程碑签核、豁免复核。

禁区：只读分析 + 书面记录，不改代码。

实例隔离（硬规则）：同一模块的 DE 与 DV 必须不同实例；arch 与 rev 分实例； 任务卡只准共享 spec、接口文件、条目 ID，禁止粘贴其他实例的推理过程——切断共模错误传播。 隔离切不断"同模型误读同一 spec"，故配套：歧义前置登记 + rev 锚定 spec 审查。

II.RTL 设计

PPA-Lite：APB 3.0 从机挂载的单帧包处理加速器。软件经 APB 把一帧包（4B 头 + ≤28B payload） 写进片上 SRAM，触发 start 后算法核逐 word 读出、做三类格式检查并计算 payload 摘要（sum/XOR）， 结果与错误标志经 CSR 读回，可选中断上报。

⚠ 编号约定：模块编号（M1=apb_slave_if、M2=packet_sram、M3=packet_proc_core） 与里程碑编号（M1~M4 = Lab1~Lab4）是两套体系。里程碑 M1 交付的是模块 M1+M2； 里程碑 M2 交付模块 M3；里程碑 M3 = 顶层 ppa_top 集成；里程碑 M4 = 全系统回归与覆盖率。

包格式与地址空间（spec §3 §4.2）

一帧 = 固定 4B 头 + 可变 payload，APB 按 word 写入 0x040~0x05C 窗口
pkt_lenByte0 · 总包长∈[4,32]
pkt_typeByte1 · one-hot 01/02/04/08
flagsByte2 · 保留=0x00
hdr_chkByte3 · B0^B1^B2
payload…Byte4~(N-1) · ≤28B · 摘要 sum(8bit 截断) / XOR
地址区域说明
0x000–0x028CSR 区11 个寄存器（下表）；地址常量唯一定义于 tb/uvm/env/ppa_reg_defs.sv
0x040–0x05CPKT_MEM 窗口8 word 映射 SRAM 写端口；busy=1 期间写被拒（PSLVERR=1）；APB 读该窗返回占位 0（r7，无读回通路）
其余保留访问 → PSLVERR=1，无副作用（§8.3 统一响应）

CSR 寄存器表（spec §5.2）

偏移寄存器关键位域属性语义
0x000CTRLenable / startRW / W1Pstart 仅 enable=1 && busy=0 时接受 → 单拍 start_o 脉冲，读回恒 0
0x004CFGalgo_mode / type_mask[7:4]RWalgo_mode=1 才做 hdr_chk 校验；type_mask 按位允许 pkt_type
0x008STATUSbusy / done / error / format_okROdone 直连模块M3 done_o 电平；下次合法 start 清结果位
0x00CIRQ_ENdone_irq_en / err_irq_enRW中断使能
0x010IRQ_STAdone_irq / err_irqRW1Cdone_i 上升沿同拍置位；写 1 清零；irq_o = done_irq | err_irq 组合输出
0x014PKT_LEN_EXPexp_pkt_len[5:0]RW0=未配置哨兵跳过比对（r4）；非 0 且 ≠pkt_len → length_error
0x018–0x024RES_*len / type / sum / xorRO模块M3 处理结果直通读回
0x028ERR_FLAGlength / type / chk_errorRO三类错误并行判定、可同时成立（§9.2），不因一类中止其他检查

III.UVM 验证

UVM-1.2 环境，一类一文件、*_pkg.sv 汇总 include。模块M3 未交付期间，其对模块M1 的结果输入 和对模块M2 的读口由受控 stub（m3_stub_if + UVM 组件 m3_stub_driver）代行—— 不是写死的 initial block，场景可编程驱动。
